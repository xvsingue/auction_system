# 🎓 毕业答辩演示操作手册

## ⚠️ 准备工作 (答辩前 5 分钟必做)

1.  **启动 Redis 服务**: 确保终端输入 `redis-cli ping` 返回 `PONG`。
2.  **开启三个终端窗口** (确保都在 `venv` 环境下):(cd /Users/apple/auction_system   source venv/bin/activate)
    * **窗口 1 (Django)**: `python manage.py runserver`
    * **窗口 2 (Worker)**: `celery -A auction_backend worker -l info  或者 python -m celery -A auction_backend worker -l info`
    * **窗口 3 (Beat)**: `python -m celery -A auction_backend beat -l info`
3.  **清空/重置数据 (可选)**: 如果之前的测试数据太乱，建议去后台把过期的拍卖场次删掉，或者新建几个全新的。

## 🎬 演示流程

### 第一幕：卖家发布与管理员审核 (展示业务流程)
1.  **登录卖家**: 使用 `seller1` 登录。
2.  **发布拍品**:
    * 点击右上角 **“+ 发布拍品”**。
    * 填写名称（如“传世青花瓷”），起拍价 1000，上传图片。
    * 提交后，页面跳转至 **“我的库存”**，指出状态为 **“待审核”**。
3.  **切换管理员**: 退出卖家，登录 `admin`。
    * 系统自动跳转至 **“管理控制台” (Dashboard)**。
    * 介绍左侧导航栏和右侧数据大屏（ECharts）。
4.  **审核与排期**:
    * 点击左侧 **“拍品审核”** -> 找到“传世青花瓷” -> 修改状态为 **“待开拍”** -> 保存。
    * 点击左侧 **“拍卖排期”** -> 增加场次 -> 关联该拍品。
    * **关键设置**: 开始时间设为**当前时间**，结束时间设为 **2分钟后** (为了演示自动结束)，状态设为 **“进行中”**。

### 第二幕：买家实时竞价 (展示核心亮点)
1.  **登录买家**: 退出管理员，登录 `buyer2` (或注册新号)。
2.  **进入大厅**: 登录后直接展示 **“拍卖大厅”**。
3.  **实时竞价**:
    * 点击刚才发布的商品，进入详情页。
    * **亮点话术**: “大家可以看到，这里展示了拍品详情。系统支持无刷新实时更新。”
    * 输入金额 `1100`，点击 **“立即出价”**。
    * 展示下方出价记录实时出现 `buyer2`。
    * *(进阶演示)*: 如果有条件，开一个隐身窗口登录另一个账号出价，展示当前页面价格自动跳变。

### 第三幕：全自动结算 (展示技术深度)
1.  **等待结束**:
    * 回到详情页，指着倒计时或时间说：“为了演示 Celery 自动化，我设置了较短的结束时间。”
    * **切换到 PyCharm 终端**: 展示 **Beat** 和 **Worker** 窗口的日志滚动。
    * **亮点话术**: “系统后台的 Celery Beat 正在毫秒级监控，一旦时间到，Worker 会自动进行结算。”
2.  **见证成交**:
    * 当终端打印 `Sending due task...` 和 `成交！` 后。
    * 刷新前端页面，状态变为 **“已结束”**。
    * 解释：系统已自动判断赢家并生成了订单。

### 第四幕：支付与资金流 (展示闭环)
1.  **查看订单**: 点击右上角头像 -> **“我的订单”**。
2.  **支付**:
    * 看到刚才生成的订单，点击 **“去支付”**。
    * 输入密码（`123456`），点击确认。
    * 提示支付成功。
3.  **查看余额**:
    * 点击头像 -> **“个人中心”**。
    * 展示余额已扣除，下方流水表中增加了一条“支付”记录。

### 第五幕：管理员全局监控 (收尾)
1.  **切回管理员**: 登录 `admin`。
2.  **数据大屏**:
    * 展示首页的 ECharts 图表（分类占比、交易趋势）。
    * 展示左侧的“资金流水”模块，证明平台确实扣了手续费。
3.  **总结**: “至此，一个包含发布、竞价、风控、结算的完整拍卖闭环演示完毕。”

## ❓ 常见问题救急 (FAQ)

* **Q: 图片不显示/白板？**
    * A: 检查 `settings.py` 里的 `DEBUG = True` 是否开启，或者检查 `static` 文件夹下是否有 `js/vue.min.js` 等文件。
* **Q: 自动结算没反应？**
    * A: 检查终端 3 (Celery Beat) 是否正在运行？是否不小心关掉了？
* **Q: 支付提示余额不足？**
    * A: 去“个人中心”点一下那个白色的 **“模拟充值”** 按钮，瞬间到账 1万。
